## PROJECT PURPOSE

Provide reinforcement-learning researchers with a high-throughput, vectorized Gymnasium environment for the supported Super Mario Bros NES mapper 0/NROM workload while preserving deterministic lane-isolated execution, training and playback compatibility, and reproducible performance evaluation.

## PROJECT REQUIREMENTS

- Support only Super Mario Bros NES on mapper 0/NROM unless broader emulator scope is deliberately added and validated without regressing the specialized workload.
- Keep ROM content out of the repository and distributions, allow users to import it through module and installed command entry points and run without supplying its path through the Stable Retro-compatible `RETRO_DATA_PATH` layout, and require canonical validation and performance comparisons to use ROM SHA-256 `f61548fdf1670cffefcc4f0b7bdcdd9eaba0c226e3b74f8666071496988248de`.
- Preserve the public `supermariobrosnes_turbo` package, `SuperMarioBrosNesTurboVecEnv`, `Actions`, metadata-derived action meanings and sets, state helpers, and manual and policy playback entry points.
- Conform to the Gymnasium `VectorEnv` reset and step return contracts, permanently report disabled autoreset, block further stepping of terminal lanes, and support selective reset through `options["reset_mask"]`.
- Keep every vector lane deterministic under seeding and independently emulated; resetting selected lanes must leave every unselected lane's emulator state, random stream, observation history, sticky action, and counters unchanged.
- Allow live vector lanes to be captured and selected lanes to be restored from reusable same-instance snapshots without advancing emulation or changing unselected lanes.
- Preserve `Actions.ALL`, `Actions.FILTERED`, Stable Retro-compatible 36-way `Actions.DISCRETE`, `Actions.MULTI_DISCRETE`, and exact named or caller-supplied button-label action tables through `use_restricted_actions`, together with grayscale or RGB observations, frame skip, optional max-pooling, crop removal or masking, resize, frame stacking, and CHW or HWC layouts.
- Preserve constructor-time initialization from one packaged or named state, path, or byte payload; expose an immutable ordered catalog containing every saved state selectable by an environment instance; reset selected lanes to explicit catalog indices with index zero as the deterministic default; report active indices; and keep state sampling, reset timing, and lane routing outside the provider, including for packaged states from `Level1-1` through `Level1-4`.
- Preserve native game-over and flag-completion termination and expose raw lives, level, score, time, position, and scrolling signals so downstream systems can own additional reward shaping, task events, termination, and outcomes; expose opt-in research signals only as documented semantic values and never return unrequested game-state keys.
- Keep `train.py [<State>]` and `play.py [<State>]`, together with installed command equivalents, as an exact discoverable-state-keyed action-run workflow whose default algorithm is Go-Explore: all trainers must default to the `standard` action set; training without a state must process all 32 canonical levels in game order, save one independently playable policy per level, and report overall level progress; playback without a state must start from `Level1-1` and must allow an explicitly uncapped rate without intentional delay or display-sync throttling; policies must represent positive-duration action runs without flat-sequence compatibility; Go-Explore must perform trajectory finding without robustification and emit the same playable action-run policy contract as beam search; beam search and Go-Explore must rank raw game-score gains ahead of every episode-length difference and use fewer steps to break equal-score ties; default Go-Explore training must write to and replace one deterministic state-only destination without an algorithm suffix while custom or non-default runs must refuse to overwrite existing policies unless explicitly forced; trainers must continue through the transition budget after completion by default while allowing an explicit first-completion stop, lock successful candidates against eviction, and select completed candidates only by shaped episode return; playback must treat an explicitly supplied state as the start state, automatically use the matching canonical policy whenever gameplay enters a canonical level, prefer beam over JERK when only legacy algorithm-specific policies exist, and not require a neural-network framework.
- Keep trainer behavior task-general across supported states: trainers must not branch on state identity to alter reward shaping or inject, constrain, seed, rank, or replace policy behavior with hand-authored action sequences or other solution-specific logic.
- Keep the public `step()` path throughput-oriented across batched emulation, native reward and termination, preprocessing, frame stacking, and Gymnasium infos; performance claims must evaluate that path with matched workloads and statistically validated, reproducible comparisons, and neutral feature changes must retain at least a `0.99` median candidate-to-baseline ratio with a bootstrap 95% lower bound of at least `0.98`.
- Preserve the `cp39-abi3` extension contract for supported platforms and validate releases on CPython 3.14 so one wheel per platform remains compatible with supported Python versions.
